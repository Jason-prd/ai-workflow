## 二、技术栈选型

### 2.1 技术栈清单

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| Web框架 | FastAPI | >=0.100 | 高性能异步框架，自带OpenAPI |
| ORM | SQLAlchemy | >=2.0 | Python ORM，支持异步 |
| 数据库 | SQLite | 3.x | MVP轻量方案，后续可迁移PostgreSQL |
| AI框架 | LangChain | >=0.1 | LLM应用框架，简化AI节点开发 |
| 任务调度 | APScheduler | >=3.10 | CRON定时触发 |
| 验证 | Pydantic | >=2.0 | 数据校验 |
| 异步任务 | asyncio | 内置 | 节点异步执行 |
| 日志 | Python logging | 内置 | 执行日志记录 |

### 2.2 技术选型理由

1. **FastAPI**
   - 异步高性能，适合IO密集型工作流
   - 自动生成API文档，降低前后端沟通成本
   - 类型提示友好，便于维护

2. **LangChain**
   - 封装主流LLM provider，减少集成工作量
   - 提供PromptTemplate、Chain等抽象
   - 工具( Tools )机制天然适合节点扩展

3. **SQLite**
   - MVP阶段零部署成本
   - 支持JSON字段，存储工作流DSL灵活
   - 后续可无缝迁移到PostgreSQL

4. **APScheduler**
   - 纯Python实现，依赖少
   - 支持CRON表达式
   - 可持久化任务到数据库

## 五、核心模块设计

### 5.1 工作流引擎 (WorkflowEngine)

**职责**：解析工作流图结构，管理执行流程

```python
class WorkflowEngine:
    async def execute(workflow_id: int, inputs: dict) -> ExecutionResult:
        """执行工作流"""
        
    async def validate_graph(graph_data: dict) -> bool:
        """验证图结构合法性"""
        
    def get_execution_order(nodes: list, edges: list) -> list:
        """获取拓扑排序后的执行顺序"""
```

**核心逻辑**:
1. 解析graph_data，获取节点和边
2. 进行拓扑排序，确定执行顺序
3. 依次调用节点执行器执行各节点
4. 传递上下文数据（上一个节点的输出作为下一个的输入）
5. 收集执行结果，更新执行日志

### 5.2 节点执行器 (NodeExecutor)

**职责**：根据节点类型调用对应的工具

```python
class NodeExecutor:
    async def execute(node: Node, context: dict) -> NodeResult:
        """执行单个节点"""
        
    def register_handler(node_type: str, handler: Callable):
        """注册节点处理器"""
```

**内置节点处理器**:
- `llm`: LLM调用节点 (LangChain)
- `http`: HTTP API请求节点
- `feishu_message`: 飞书消息发送
- `feishu_doc`: 飞书文档读取
- `condition`: 条件分支节点
- `start/end`: 开始/结束节点

### 5.3 触发器调度器 (TriggerScheduler)

**职责**：管理工作流的定时触发

```python
class TriggerScheduler:
    def start():
        """启动调度器"""
        
    def add_cron_job(workflow_id: int, cron_expr: str):
        """添加CRON任务"""
        
    def remove_job(workflow_id: int):
        """移除任务"""
        
    def trigger_workflow(workflow_id: int):
        """触发工作流执行"""
```

### 5.4 工具适配层 (ToolAdapter)

**职责**：封装外部工具的调用

```python
# LangChain Tool 定义示例
class LLMAdapter:
    def __init__(self, config: ToolConfig):
        self.llm = ChatOpenAI(
            model=config.model,
            api_key=config.api_key
        )
        
    async def invoke(prompt: str, context: dict) -> str:
        """调用LLM生成内容"""

class FeishuAdapter:
    async def send_message(chat_id: str, content: str):
        """发送飞书消息"""
        
    async def get_doc_content(doc_id: str):
        """获取飞书文档内容"""

class HTTPAdapter:
    async def request(method: str, url: str, headers: dict, body: any):
        """发起HTTP请求"""
```

## 六、部署方案

### 6.1 开发环境

```bash
# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 启动服务
uvicorn app.main:app --reload --port 8000
```

### 6.2 Docker 部署

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

```yaml
# docker-compose.yml
version: '3.8'

services:

      - "8000:8000"
    volumes:
      - ./data:/app/data

      - DATABASE_URL=sqlite:///./data/workflow.db
```

## 七、安全考虑

1. **认证**: JWT Token 认证
2. **鉴权**: 用户只能操作自己的数据
3. **敏感信息**: 工具配置加密存储 (AES)
4. **输入校验**: Pydantic 严格校验
5. **限流**: 防止恶意触发 (每个用户每分钟最多10次)
