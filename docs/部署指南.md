## 一、环境变量参考

### 应用配置

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `DEBUG` | 否 | false | 调试模式（生产必须为 false） |
| `DATABASE_URL` | 否 | sqlite:///./ai_workflow.db | 数据库连接地址 |
| `SECRET_KEY` | **是** | — | JWT 签名密钥（**生产必须设置**，建议 64+ 随机字符） |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | 否 | 1440 | Access Token 过期时间（分钟） |
| `MAX_NODES_PER_WORKFLOW` | 否 | 20 | 单工作流最大节点数 |
| `MAX_CONCURRENT_EXECUTIONS` | 否 | 5 | 最大并发执行数 |
| `EXECUTION_TIMEOUT_SECONDS` | 否 | 60 | 执行超时时间（秒） |
| `MAX_EXECUTION_LOGS` | 否 | 100 | 最大执行日志保留数 |
| `LOG_RETENTION_DAYS` | 否 | 7 | 执行日志保留天数 |

### OpenAI

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `OPENAI_API_KEY` | **是** | — | OpenAI API Key |
| `OPENAI_MODEL` | 否 | gpt-4o-mini | OpenAI 模型 |
| `OPENAI_MAX_TOKENS` | 否 | 2000 | 最大 Token 数 |
| `OPENAI_TEMPERATURE` | 否 | 0.7 | 生成温度参数 |

### 飞书集成

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `FEISHU_APP_ID` | **是** | — | 飞书自建应用 App ID（格式：cli_xxx） |
| `FEISHU_APP_SECRET` | **是** | — | 飞书自建应用 App Secret |
| `FEISHU_VERIFICATION_TOKEN` | **是** | — | 飞书事件订阅验证 Token |
| `FEISHU_ENCRYPT_KEY` | 否 | — | 飞书事件加密密钥 |
| `FEISHU_CALENDAR_POLL_INTERVAL_SECONDS` | 否 | 60 | 日历轮询间隔（秒） |
| `FEISHU_CALENDAR_REMIND_MINUTES` | 否 | 5 | 日历提前提醒分钟数 |

## 二、Docker Compose 部署

### 2.1 前置条件

- Docker 20.10+
- Docker Compose 2.0+
- 域名（可选，用于 HTTPS）

### 2.2 目录结构

```
backend/
├── Dockerfile
├── docker-compose.yml
├── nginx.conf
├── .env.production      # 复制自 .env.production
├── ssl/                  # SSL 证书目录（可选）
│   ├── fullchain.pem
│   └── privkey.pem
└── data/                 # 数据持久化目录
```

### 2.3 快速部署

```bash
cd backend

# 1. 复制环境变量文件
cp .env.production .env
# 编辑 .env，填写所有必填变量

# 2. 创建数据目录
mkdir -p data

# 3. 构建并启动服务
docker compose up -d --build

# 4. 查看服务状态
docker compose ps

# 5. 查看日志
docker compose logs -f api
```

### 2.4 生产 .env 示例

```bash
# === 必须修改 ===
DEBUG=false

# JWT（必须使用强随机密钥！）
SECRET_KEY=your-super-secret-key-minimum-64-characters-long-xxxxxxxxxxxxxxxx

# OpenAI
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
OPENAI_MODEL=gpt-4o-mini

# 飞书（从飞书开放平台获取）
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_VERIFICATION_TOKEN=your_verification_token

# 工作流执行配置
MAX_CONCURRENT_EXECUTIONS=5
EXECUTION_TIMEOUT_SECONDS=60
```

### 2.5 服务启停

```bash
# 启动所有服务
docker compose up -d

# 停止所有服务
docker compose down

# 重启所有服务
docker compose restart

# 重启 API 服务
docker compose restart api

# 重新构建并启动
docker compose up -d --build
```

### 2.6 前端构建产物部署

```bash
# 1. 构建前端
cd ../frontend
npm run build

# 2. 将构建产物复制到 backend 目录
cp -r dist ../backend/frontend-dist

# 3. 重新启动 Nginx
docker compose restart nginx
```

### 2.7 扩展部署

```bash
# 扩展 API 服务实例（需要配置 Nginx upstream）
docker compose up -d --scale api=2

# 注意：SQLite 不支持多实例写入，生产环境建议使用 PostgreSQL
# DATABASE_URL=postgresql+asyncpg://user:pass@host:5432/ai_workflow
```

## 三、Nginx 配置

### 3.1 配置文件说明

主要配置文件位于 `backend/nginx.conf`，包含：

- **上游服务**：指向 Docker 内部网络中的 `api:8000`
- **健康检查端点**：`/health` 直接代理到 API
- **Gzip 压缩**：启用 JSON/CSS/JS 等资源压缩
- **Keepalive 连接**：减少与后端的连接建立开销
- **静态文件缓存**：前端资源 30 天缓存

### 3.2 路由规则

| 路径 | 路由到 | 说明 |
|------|--------|------|
| `/health` | API | 健康检查 |
| `/api/*` | API | 后端 API |
| `/docs` | API | OpenAPI 文档 |
| `/redoc` | API | ReDoc 文档 |
| `/static/*` | 静态文件 | 前端静态资源 |
| `/*` | 前端 SPA | 其余路由到 index.html |

### 3.3 HTTPS 配置（生产环境）

1. 申请 SSL 证书（如 Let's Encrypt）：

```bash
# 使用 certbot 申请证书
certbot certonly --nginx -d your-domain.com
```

2. 将证书放入 `backend/ssl/` 目录：
```
backend/ssl/fullchain.pem
backend/ssl/privkey.pem
```

3. 取消 `nginx.conf` 中 HTTPS Server 块的注释

### 3.4 Nginx 性能调优

```nginx
# worker 进程数（设为 CPU 核心数）
worker_processes auto;

# 每个 worker 的连接数
worker_connections 2048;

# 超时配置
proxy_read_timeout 60s;
proxy_connect_timeout 10s;

# Gzip 压缩
gzip on;
gzip_types application/json;
gzip_min_length 1000;

# 启用连接缓存
upstream backend {
    server 127.0.0.1:8000;
    keepalive 64;
}
```

docker compose logs api --tail=100      # 最后 100 行

docker compose logs nginx

```bash

docker exec -it ai-workflow-nginx /bin/sh

数据存储在 `backend/data/` 目录：

```

└── ai_workflow.db    # SQLite 数据库文件

**重要**：生产环境建议：
1. 定期备份数据库文件
2. 考虑迁移到 PostgreSQL 以获得更好的并发支持

```bash

docker compose down

docker compose down -v

# 清理未使用的镜像
docker image prune -f

# 完全重建
docker compose down --rmi local
docker compose up -d --build
```

## 七、性能调优

### 7.1 API 服务调优

**增加 Uvicorn Workers**：

修改 `Dockerfile` 中的启动命令：
```dockerfile
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "4"]
```

**注意**：SQLite 不支持多 worker 写入，生产环境请使用 PostgreSQL。

### 7.2 并发配置

```bash
# 在 .env 中调整
MAX_CONCURRENT_EXECUTIONS=5      # 最大并发执行数
EXECUTION_TIMEOUT_SECONDS=60      # 单次执行超时
```

### 7.3 Nginx 调优

```nginx
# 增加 worker 连接数
worker_connections 2048;

# 启用连接缓存
upstream backend {
    server 127.0.0.1:8000;
    keepalive 64;
}

# 缓存静态资源
location /static {
    alias /usr/share/nginx/html/static;
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

### 7.4 OpenAI 响应优化

```bash
# 减少 Token 限制以加快响应
OPENAI_MAX_TOKENS=1000

# 降低 Temperature 以获得更确定性输出
OPENAI_TEMPERATURE=0.3
```

## 八、备份与恢复

### 8.1 备份

```bash
# 备份 SQLite 数据库
mkdir -p data/backup
cp data/ai_workflow.db data/backup/ai_workflow_$(date +%Y%m%d).db

# 备份到外部存储
tar -czf backup_$(date +%Y%m%d).tar.gz data/ backend/.env
```

### 8.2 恢复

```bash
# 停止服务
docker compose stop api

# 恢复数据库
cp data/backup/ai_workflow_20260404.db data/ai_workflow.db

# 重启服务
docker compose start api
```

### 8.3 自动化备份（crontab）

```bash
# 每天凌晨 3 点备份
0 3 * * * cp /path/to/data/ai_workflow.db /backup/ai_workflow_$(date +\%Y\%m\%d).db

# 保留最近 7 天备份
0 3 * * * find /backup -name "ai_workflow_*.db" -mtime +7 -delete
```

## 九、常见问题

### Q1: Docker 镜像构建失败

**症状**：`ModuleNotFoundError` 或 `pip install failed`

**解决方案**：
```bash
# 1. 清理 Docker 缓存，重新构建
docker build --no-cache -t ai-workflow-api .

# 2. 检查 requirements.txt 是否有语法错误
pip install -r requirements.txt --dry-run

# 3. 使用国内镜像（国内服务器）
RUN pip install --no-cache-dir -r requirements.txt \
    -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q2: Nginx 502 Bad Gateway

**原因**：
1. API 服务未启动
2. API 服务端口配置错误
3. 超时配置过短

**解决方案**：
```bash
# 1. 确认 API 服务运行中
curl http://localhost:8000/health

# 2. 检查 Docker 网络
docker network inspect ai-workflow_ai-workflow-network

# 3. 检查 Nginx 日志
docker compose logs nginx | grep -i error
```

### Q3: 飞书机器人无法收发消息

**原因**：
1. Webhook 回调 URL 不通
2. 签名验证失败
3. 应用权限未开通

**解决方案**：
```bash
# 1. 验证 Webhook URL 可访问
curl -X GET https://your-domain.com/health

# 2. 检查飞书应用权限（开放平台 → 应用 → 权限管理）
# 需要：im:message, calendar:calendar 等

# 3. 确认 FEISHU_VERIFICATION_TOKEN 正确配置

# 4. 检查环境变量是否正确注入容器
docker compose exec api env | grep FEISHU
```

### Q4: 执行超时

**原因**：
1. OpenAI API 响应慢
2. 网络延迟
3. 节点数量过多

**解决方案**：
```bash
# 1. 增加执行超时时间
EXECUTION_TIMEOUT_SECONDS=120

# 2. 减少工作流节点数
MAX_NODES_PER_WORKFLOW=10

# 3. 检查 OpenAI API 延迟
```

### Q5: 数据库连接问题

**SQLite 生产环境注意**：
SQLite 不支持多并发写入，生产环境建议迁移到 PostgreSQL。

```bash
# 迁移到 PostgreSQL
# 1. 导出数据
sqlite3 ai_workflow.db ".dump" > backup.sql

# 2. 修改 DATABASE_URL
DATABASE_URL=postgresql+asyncpg://user:pass@host:5432/ai_workflow

# 3. 重启服务
docker compose restart api
```

### Q6: 如何水平扩展？

**当前限制**：SQLite 不支持水平扩展。

**PostgreSQL 方案**：
```bash
# 1. 使用 PostgreSQL 数据库
DATABASE_URL=postgresql+asyncpg://user:pass@pghost:5432/ai_workflow

# 2. 增加 API workers（需要 Gunicorn 等）
# 修改 Dockerfile 使用 gunicorn
CMD ["gunicorn", "app.main:app", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "-b", "0.0.0.0:8000"]

# 3. 多实例部署
docker compose up -d --scale api=4
```

### Q7: 日志文件在哪？

```bash
# API 日志（stdout/stderr）
docker compose logs api

# Nginx 访问日志
docker compose logs nginx

# 进入容器查看
docker exec ai-workflow-api cat /var/log/uvicorn.log
```
