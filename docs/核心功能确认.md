## 一、MVP核心功能列表

基于项目分析文档，经过优先级评估，确定以下5个核心功能：

### 1. 工作流编排引擎
- **功能描述**：可视化拖拽式工作流设计器，支持节点添加、连接、删除，支持条件分支和串行执行
- **用户价值**：用户无需编码即可设计自动化流程，降低使用门槛
- **优先级**：P0（必须做）

### 2. AI任务节点
- **功能描述**：内置LLM调用节点，支持提示词模板配置、上下文管理、单模型调用
- **用户价值**：实现AI能力即插即用，是产品的核心差异化价值
- **优先级**：P0（必须做）

### 3. 触发器
- **功能描述**：支持定时触发（CRON）、手动触发两种方式
- **用户价值**：让工作流能够自动/按需执行，实现真正的自动化
- **优先级**：P0（必须做）

### 4. 执行日志
- **功能描述**：完整记录工作流执行过程，展示每个节点的输入输出和执行状态
- **用户价值**：支持调试和错误追溯，是运维的刚需
- **优先级**：P0（必须做）

### 5. 基础工具集成
- **功能描述**：预置飞书（消息/文档）、HTTP API两个基础连接器
- **用户价值**：覆盖主流场景，让工作流能够与外部系统交互
- **优先级**：P1（可后续做，但MVP需包含）

## 三、MVP用户交互流程

```
1. 注册/登录 → 2. 创建工作流 → 3. 拖拽添加节点 → 4. 配置AI提示词 → 5. 设置触发器 → 6. 测试运行 → 7. 启用
```

## 四、MVP技术范围

- **后端**：Python + FastAPI + LangChain
- **前端**：React + React Flow
- **数据库**：SQLite（MVP轻量）
- **部署**：本地运行 / Docker
