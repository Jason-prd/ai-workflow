## 一、产品概述

### 1.1 产品定位

**AI自动化工作流** 是一款面向企业和个人用户的低代码AI自动化平台，通过可视化拖拽方式，用户无需编程即可创建由AI驱动的自动化工作流，解决重复性工作效率低下、跨系统数据孤岛等问题。

### 1.2 产品愿景

让AI能力触手可及，让重复工作自动完成。

### 1.3 核心价值主张

| 价值点 | 说明 |
|--------|------|
| **低门槛** | 可视化拖拽设计，无需编码即可创建AI工作流 |
| **高效率** | 自动化执行重复任务，释放人力投入高价值工作 |
| **可扩展** | 预置主流工具连接器，支持API自定义扩展 |
| **可追溯** | 完整执行日志，实时监控运行状态 |

### 1.4 MVP范围

- **用户规模**：支持个人用户、小团队（1-10人）
- **功能范围**：5个核心功能（工作流编排引擎、AI任务节点、触发器、执行日志、基础工具集成）
- **工具集成**：飞书（消息/文档）、HTTP API
- **部署方式**：本地运行 / Docker

## 三、功能详细说明

### 3.1 功能一：工作流编排引擎

#### 3.1.1 功能描述

可视化拖拽式工作流设计器，支持节点添加、连接、删除，支持条件分支和串行执行。

#### 3.1.2 功能详情

| 功能点 | 说明 |
|--------|------|
| **画布** | 无限画布，支持缩放、平移 |
| **节点** | 触发器节点、AI处理节点、工具节点、条件节点 |
| **连线** | 节点间连线，支持单向流转 |
| **保存/发布** | 保存草稿、发布上线 |

#### 3.1.3 节点类型

| 节点类型 | 用途 | 配置项 |
|----------|------|--------|
| 触发器 | 工作流启动入口 | 定时/手动 |
| AI任务 | 调用LLM处理内容 | 模型选择、提示词、输入变量 |
| 工具执行 | 调用外部工具 | 工具类型、API配置 |
| 条件分支 | 根据条件判断流转 | 条件表达式 |

#### 3.1.4 交互设计

```
拖拽节点 → 连接节点 → 配置参数 → 保存/发布
```

#### 3.1.5 MVP约束

- 仅支持串行执行，暂不支持并行/循环
- 最多20个节点/工作流
- 仅支持单条件分支

---

### 3.2 功能二：AI任务节点

#### 3.2.1 功能描述

内置LLM调用节点，支持提示词模板配置、上下文管理、单模型调用。

#### 3.2.2 功能详情

| 功能点 | 说明 |
|--------|------|
| **模型选择** | 支持OpenAI GPT系列（GPT-4o、GPT-4o-mini） |
| **提示词模板** | 支持变量占位符，如 {{input}}、{{name}} |
| **上下文管理** | 自动维护对话历史（最近3轮） |
| **输出配置** | 定义输出变量，供下游节点使用 |

#### 3.2.3 配置参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 模型 | 下拉选择 | 是 | 选择LLM模型 |
| 提示词 | 文本 | 是 | 支持变量替换 |
| 输入变量 | 变量引用 | 是 | 上游节点输出 |
| 输出变量 | 字符串 | 是 | 供下游引用 |
| Temperature | 数值 | 否 | 0-2，默认0.7 |
| Max Tokens | 数值 | 否 | 最大生成token数 |

#### 3.2.4 MVP约束

- 仅支持OpenAI API
- 仅支持单模型调用
- 上下文窗口限制：最多3轮历史

---

### 3.3 功能三：触发器

#### 3.3.1 功能描述

支持定时触发（CRON）、手动触发两种方式。

#### 3.3.2 触发器类型

| 触发类型 | 说明 | 配置项 |
|----------|------|--------|
| **手动触发** | 用户点击执行按钮 | 无需配置 |
| **定时触发** | 按CRON表达式定时执行 | CRON表达式、时区 |

#### 3.3.3 CRON配置

| 字段 | 说明 | 示例 |
|------|------|------|
| 分钟 | 0-59 | * 每分钟 |
| 小时 | 0-23 | * 每小时 |
| 日期 | 1-31 | 1,15 每月1号和15号 |
| 月份 | 1-12 | * 每月 |
| 星期 | 0-6 | 1-5 工作日 |

**常用CRON示例**：
- `0 9 * * 1-5` → 每周一到周五9:00执行
- `0 0 * * *` → 每天0点执行
- `0 */2 * * *` → 每2小时执行

#### 3.3.4 MVP约束

- 仅支持手动触发和定时触发
- 暂不支持Webhook、事件触发

---

### 3.4 功能四：执行日志

#### 3.4.1 功能描述

完整记录工作流执行过程，展示每个节点的输入输出和执行状态。

#### 3.4.2 功能详情

| 功能点 | 说明 |
|--------|------|
| **执行记录** | 每次执行生成一条记录 |
| **节点状态** | 显示每个节点的执行状态（成功/失败/运行中） |
| **输入输出** | 展示每个节点的输入参数和输出结果 |
| **耗时统计** | 显示每个节点执行耗时 |
| **错误信息** | 失败节点显示详细错误原因 |

#### 3.4.3 日志字段

| 字段 | 说明 |
|------|------|
| 执行ID | 唯一标识 |
| 工作流 | 所属工作流名称 |
| 触发方式 | 手动/定时 |
| 开始时间 | 执行开始时间 |
| 结束时间 | 执行结束时间 |
| 状态 | 成功/失败/运行中 |
| 节点详情 | 每个节点的输入/输出/状态/耗时 |

#### 3.4.4 MVP约束

- 仅保留最近100条执行记录
- 日志保留7天

---

### 3.5 功能五：基础工具集成

#### 3.5.1 功能描述

预置飞书（消息/文档）、HTTP API两个基础连接器。

#### 3.5.2 飞书集成

| 功能 | 说明 | 配置项 |
|------|------|--------|
| **发送消息** | 发送文本/卡片消息到飞书群 | 群ID、消息内容 |
| **创建文档** | 在飞书创建云文档 | 文档标题、内容 |
| **读取文档** | 读取飞书云文档内容 | 文档链接/ID |

**飞书配置**：
- 需要飞书企业应用权限（应用ID + 应用密钥）
- 需要获取群聊ID

#### 3.5.3 HTTP API集成

| 功能 | 说明 | 配置项 |
|------|------|--------|
| **HTTP请求** | 发起GET/POST请求 | URL、Method、Headers、Body |

**HTTP请求配置**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| URL | 字符串 | 是 | 请求地址 |
| Method | 下拉 | 是 | GET/POST/PUT/DELETE |
| Headers | JSON | 否 | 请求头 |
| Body | JSON | 否 | 请求体（POST/PUT） |
| 超时 | 数值 | 否 | 默认30秒 |

#### 3.5.4 MVP约束

- 仅支持飞书消息、文档、HTTP API
- 不支持认证授权（API Key、Bearer Token）

## 六、非功能需求

### 6.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| **页面加载** | < 2秒 | 首屏加载时间 |
| **API响应** | < 500ms | 常规接口响应时间 |
| **工作流执行** | < 60秒 | 单次执行超时时间 |
| **并发执行** | 5个/用户 | 支持同时运行的工作流数 |

### 6.2 可用性需求

| 指标 | 要求 |
|------|------|
| **服务可用性** | 99.5% |
| **数据持久化** | 执行日志保留7天 |
| **错误恢复** | 失败节点自动重试1次 |

### 6.3 安全需求

| 需求 | 说明 |
|------|------|
| **身份认证** | 用户名+密码登录，Session管理 |
| **数据隔离** | 用户数据完全隔离 |
| **敏感信息** | API Key等敏感信息加密存储 |
| **日志审计** | 记录操作日志 |

### 6.4 兼容性需求

| 需求 | 说明 |
|------|------|
| **浏览器** | Chrome 90+、Edge 90+、Firefox 90+、Safari 15+ |
| **响应式** | 支持桌面端（1024px+） |

### 6.5 扩展性需求

| 需求 | 说明 |
|------|------|
| **插件化** | 工具节点采用插件化架构，便于扩展 |
| **API开放** | 预留RESTful API接口供二次开发 |

## 七、术语表

| 术语 | 定义 |
|------|------|
| **工作流(Workflow)** | 由多个节点组成的自动化执行流程 |
| **节点(Node)** | 工作流中的执行单元（触发器/AI任务/工具） |
| **触发器(Trigger)** | 启动工作流的条件（定时/手动） |
| **AI任务节点** | 调用LLM处理内容的节点 |
| **执行日志** | 记录工作流每次执行的详细信息 |
| **CRON** | 定时任务表达式格式 |
| **MVP** | 最小可行产品 |
