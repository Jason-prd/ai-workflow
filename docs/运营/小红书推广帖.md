## 版本二：掘金/技术社区风格

**标题：**
【开源】用 React Flow + FastAPI 做了一个低代码 AI 工作流平台

**正文：**

大家好，今天分享一个我最近做的开源项目——AI 自动化工作流平台。

**背景**

团队日常充斥着大量重复性工作：日报生成、数据汇总、跨系统数据同步。这些事情上 Zapier/Make 太贵，RPA 太重，于是决定自己动手。

**技术方案**

- **后端**：FastAPI + SQLAlchemy 2.0 + Pydantic v2 + JWT + APScheduler
- **前端**：React 18 + TypeScript + Vite + React Flow + Ant Design 6 + TailwindCSS 4
- **部署**：Docker Compose + Nginx

**核心功能实现**

```
1. 可视化画布
基于 React Flow 的无限画布，支持拖拽节点、连线、缩放、平移。
每个节点类型（触发器/AI任务/工具/条件分支）有独立的配置面板。

2. AI 节点
接入 GPT-4o-mini，支持提示词模板：
"请根据以下数据生成日报：{{input}}"
支持 temperature、max_tokens 灵活配置。

3. 触发方式
- 手动触发：点击按钮立即执行
- 定时触发：CRON 表达式，支持时区配置

4. 工具节点
- 飞书消息：发送文本/卡片消息到群聊或个人
- 飞书文档：自动创建/读取飞书云文档
- HTTP 请求：集成任意 RESTful API

5. 条件分支
支持条件表达式判断，决定工作流执行路径：
{{status}} == "active" ? 继续执行 : 终止

6. 执行日志
每个节点的输入/输出清晰可见。
执行状态实时追踪（Pending → Running → Success/Failed）。
失败节点自动重试（1次）。
```

**快速启动**

```bash
git clone https://github.com/Jason-prd/ai-workflow.git
cd ai-workflow/backend
cp .env.example .env
# 编辑 .env 填入 OPENAI_API_KEY 和飞书配置
docker-compose up -d
# 前端：http://localhost
# API 文档：http://localhost:8000/docs
```
