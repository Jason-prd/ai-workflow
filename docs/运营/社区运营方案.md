## 一、项目定位与现状分析

### 1.1 项目核心定位

**一句话描述：** 低代码可视化 AI 驱动自动化工作流平台——拖拽即可创建 AI 自动化工作流，无需编程。

**核心差异化卖点：**
- 🎨 **可视化拖拽设计**（React Flow 无限画布）
- 🤖 **AI 原生**：内置 GPT-4o/GPT-4o-mini 任务节点，提示词模板支持变量占位符
- 🔗 **企业集成**：飞书深度集成（消息、文档、日历触发器）
- ⚡ **开箱即用**：一行命令 Docker Compose 部署

### 1.2 当前痛点（需在运营中解决）

| 痛点 | 说明 | 运营对策 |
|------|------|---------|
| 0 Stars | 新仓库缺乏社会证明 | 早期核心用户社群启动 |
| 竞品林立 | n8n、Make、Zapier 已有大量用户 | 聚焦飞书生态 + 中文本地化 |
| 文档可用性 | 需持续丰富使用场景 | 分步骤场景化教程 |
| Contributors 稀缺 | 开源初期无外部贡献 | 设计友好的 first-contribution 路径 |

### 5.2 吸引 Contributors 的策略

**低门槛贡献路径设计：**

| 贡献类型 | 具体任务 | 难度 | 适合人群 |
|---------|---------|------|---------|
| 文档优化 | 补充 README 示例、修复错别字 | ⭐ | 新手 |
| 场景模板 | 提交一个工作流配置 JSON 示例 | ⭐⭐ | 初级 |
| Bug 修复 | 认领"Good First Issue" | ⭐⭐ | 初中级 |
| 新节点类型 | 开发一个新的工具节点（如 Slack） | ⭐⭐⭐ | 中级 |
| 前端组件 | 优化节点配置面板 UI | ⭐⭐ | 初中级 |
| 测试用例 | 补充单元测试覆盖率 | ⭐⭐ | 初中级 |

**具体行动：**
1. 在 Issues 中创建 5-10 个 "good first issue" 标签的问题
2. 编写 `CONTRIBUTING.md`，明确贡献流程和代码规范
3. 在 README 中增加"如何贡献"章节
4. 建立 `examples/` 目录，存放社区贡献的工作流模板
