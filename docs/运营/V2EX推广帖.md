周末肝了两周，给团队做了个低代码 AI 自动化工作流平台，先上图镇楼。

## 解决什么问题

我们团队每天都在干这些事：
- 早上把昨天数据复制给 GPT，让它生成日报
- 定时拉取飞书日历，整理成会议纪要
- 把用户反馈分门别类打标签

这些都是重复劳动，但上 RPA 太重，上 Zapier/Make 贵而且数据要走第三方。于是自己写了这个：

**GitHub: https://github.com/Jason-prd/ai-workflow**

## 核心功能

- 🎨 **可视化画布**：基于 React Flow，拖拖拽拽设计工作流，支持缩放平移
- 🤖 **AI 节点**：接入 GPT-4o-mini，支持提示词模板和 `{{变量}}` 占位符
- ⏰ **触发方式**：手动触发 + CRON 定时触发
- 🔧 **工具节点**：飞书消息、飞书文档、HTTP 请求
- 🔀 **条件分支**：支持条件表达式，自动决定执行路径
- 📊 **执行日志**：每个节点的输入输出清晰可见，失败自动重试

## 技术栈

- **后端**：FastAPI + SQLAlchemy + Pydantic + JWT + APScheduler
- **前端**：React 18 + TypeScript + Vite + React Flow + Ant Design + TailwindCSS
- **部署**：Docker Compose 一键启动，开箱即用

## 使用示例

比如这个"日报自动生成器"工作流：
1. CRON 定时触发（每天早上 9 点）
2. AI 节点根据输入数据生成日报
3. 自动发送到飞书群

只需要配置节点、连上线，就好了。

**回复区预判问题（提前准备）：**

> Q: 和 Make.com / Zapier 有什么区别？
> A: 完全开源可私有化部署，数据不经过第三方。另外我们针对飞书生态做了深度集成。

> Q: Windows 能跑吗？
> A: 能。Docker 支持 Windows/Linux/MacOS三大平台，一行命令启动。

> Q: 支持其他模型吗（Claude / 通义）？
> A: 目前主要支持 OpenAI GPT 系列，架构上支持扩展，欢迎提 PR。
