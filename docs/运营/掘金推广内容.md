## 前言

每周写周报是不是很痛苦？花 30 分钟整理数据、复制粘贴、润色措辞——这些重复性工作完全可以交给 AI 自动完成。

今天分享一个我开源的项目：AI 自动化工作流，用拖拽的方式搭建 GPT 日报生成器，5 分钟搞定，每周自动运行。

GitHub（求 Star）：https://github.com/Jason-prd/ai-workflow

## 效果预览

配置完成后，每个工作日早上 9 点：
1. AI 自动抓取今日行业动态
2. 生成格式规范的日报内容
3. 自动推送到你的飞书群

全程无需手动，真正的自动化 😎

## 什么是 AI 自动化工作流

简单来说，就是一个"低代码可视化 AI 工作流平台"：

- 🎨 可视化拖拽设计（基于 React Flow）
- 🤖 AI 原生（内置 GPT-4o/GPT-4o-mini 节点）
- 🔗 企业集成（飞书深度集成）
- ⚡ 开箱即用（Docker 一行命令部署）

## 5 分钟搭建日报生成器

### Step 1：创建工作流

打开 http://localhost/workflows，点击「新建工作流」，输入名称「AI 日报生成器」。

### Step 2：添加定时触发器

从左侧节点面板拖入一个「触发器」节点，选择「CRON 定时」。

配置 cron 表达式：`0 9 * * 1-5`（每个工作日早上 9 点执行）。

### Step 3：添加 AI 任务节点

拖入一个「AI 任务」节点，连接到触发器。

配置如下：
- 模型：GPT-4o-mini
- Prompt：
```
请根据今日热点，生成一份 {industry} 行业的简报，包含：
1. 今日要闻（3条）
2. 行业趋势（2条）
3. 值得关注的动态（1条）

要求：简洁专业，适合高管阅读，字数控制在 500 字以内。
```

### Step 4：添加飞书消息节点

拖入一个「飞书消息」节点，连接到 AI 任务节点。

配置：
- 接收人：选择目标群或指定人
- 消息内容：引用上游变量 `{{ai_result}}`

### Step 5：保存并发布

点击「保存」，然后点击「发布」。

现在每个工作日早上 9 点，AI 会自动生成日报并推送到飞书群 🎉

## 核心概念讲解

### {{variable}} 变量占位符

Prompt 里的 `{{variable}}` 是变量占位符，运行时会自动替换为上游节点的输出。

例如：`{{ai_result}}` 会自动替换为 AI 任务节点的输出内容。

### 节点连接

节点之间通过连线传递数据，上游节点的输出成为下游节点的输入。React Flow 会自动处理数据流转。

### 执行日志

每次执行都可以在「执行日志」页面查看详情，每个节点的输入、输出、耗时一目了然。

## 文章 2：技术原理

基于 React Flow 的可视化工作流设计：节点编排实战
```

**正文：**

```
## 背景

在做 AI 自动化工作流平台时，核心挑战之一是：如何设计一个易用的可视化工作流编辑器？

最终选择使用 @xyflow/react（React Flow）实现拖拽式节点编排。本文分享实践中的关键设计思路。

GitHub：https://github.com/Jason-prd/ai-workflow

## 为什么选择 React Flow

选型时对比了几个方案：
- **React Flow**：专为企业级工作流设计，API 完善，文档清晰
- **Rete.js**：老牌方案，但 API 较旧
- **自研**：成本太高，不推荐

React Flow 的优势：
- 开源免费（MIT）
- 活跃社区，持续更新
- 支持自定义节点，可灵活扩展
- 内置缩放、平移、选择等交互

## 节点类型设计

我们将节点分为四类：

### 1. 触发器节点（Trigger）
触发工作流执行，分为：
- 手动触发
- CRON 定时触发
- 飞书消息触发
- 飞书日历触发

```typescript
interface TriggerNode {
  type: 'trigger';
  triggerConfig: {
    triggerType: 'manual' | 'cron' | 'feishu_message' | 'feishu_calendar';
    config: Record<string, any>;
  };
}
```

### 2. AI 任务节点（AI Task）
执行 AI 调用：

```typescript
interface AITaskNode {
  type: 'ai_task';
  aiConfig: {
    model: 'gpt-4o' | 'gpt-4o-mini';
    prompt: string; // 支持 {{variable}} 占位符
    temperature: number;
    maxTokens: number;
  };
}
```

### 3. 工具节点（Tool）
执行具体操作：
- 飞书消息
- 飞书文档
- HTTP 请求

### 4. 条件节点（Condition）
分支逻辑判断：

```typescript
interface ConditionNode {
  type: 'condition';
  conditionConfig: {
    expression: string; // e.g. "{{status}} == 'success'"
    trueBranch: string; // 条件为真时的分支
    falseBranch: string; // 条件为假时的分支
  };
}
```

## 数据流设计

节点之间通过边（Edge）连接，数据沿边传递：

```
Trigger → AI Task → Feishu Message
              ↓
         Condition → [true branch]
                     [false branch]
```

每个节点维护自己的输入（input）和输出（output），执行引擎按拓扑顺序执行。

## 变量替换实现

Prompt 中的 `{{variable}}` 占位符在执行时替换为实际值：

```python
def replace_variables(template: str, context: dict) -> str:
    """将模板中的 {{variable}} 替换为 context 中的实际值"""
    import re
    pattern = r'\{\{(\w+)\}\}'
    
    def replacer(match):
        var_name = match.group(1)
        return str(context.get(var_name, match.group(0)))
    
    return re.sub(pattern, replacer, template)
```

## 执行引擎设计

执行引擎按以下顺序运行：

1. 从触发器节点开始
2. 按拓扑顺序遍历节点
3. 每个节点读取上游输出作为输入
4. 执行节点逻辑，产生输出
5. 将输出写入上下文，供下游使用
6. 失败节点自动重试（1次）

React Flow 非常适合这类可视化编辑器，API 设计合理，自定义节点也很方便。
